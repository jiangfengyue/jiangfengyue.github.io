Win系统下用Github或VS Code实现Unity项目版本管理

发表于 2016-12-18 | 分类于 Unity开发 | 阅读次数

Unity项目进行版本管理时，需要借助一些外部工具。目前比较流行的版本管理平台是Github。本文探讨Win系统下用Github和VS Code对Unity项目进行版本管理的方法。

1 前言

1.1 需求

VS Code与Unity的集成可见上文，本文不再赘述。通过Github进行版本管理已经非常流行，本文将就Win系统下用Github和VS Code开发Unity项目进行探讨。依托Github进行版本管理主要有两个途径，一个是使用Github的win客户端对代码直接管理，一个是通过VS Code进行集成管理。以下分述。本文默认已注册Github账户。

1.2 术语解释

  • Git：是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

  • Github：是一个面向开源及私有软件项目的托管平台，因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub。

1.3 版本说明

Win 10，Unity 5.4，VS Code1.8.0，Git 2.11.0 windows.1，Github Desktop 3.3.3.0

2 使用Github客户端进行版本管理

2.1 下载安装Github win客户端

从这里下载最新的客户端并安装。需要注意的是，这里获取的是下载器，需要二次下载安装。如果二次下载遇到错误，可以择时再试。

2.2 登陆Github账户

2.3 生成并发布Github代码仓库

点击左上角的Create按钮，选择Unity项目文件夹，点击生成，即可在本地生成仓库，并将项目文件添加到本地仓库。

点击右上角的publish按钮，将本地仓库发布到远程账户。

2.4 更新仓库

当对仓库中的文件进行增删改操作后，可以在客户端页面先admit本地提交，再通过Sync同步push到远程端。其他的操作与Github的常规操作无异，可参考相关资料。

3 使用VS Code进行版本管理

3.1 下载并安装Git

下载Git文件并安装，可以根据具体需求设置。

3.2 生成仓库

在Github网站点击Create设置新仓库，复制当前仓库的地址。

3.3 将仓库下载到本地

win+r键打开窗口，输入cmd打开命令行窗口。将当前目录调整到Unity项目目录。输入git clone 复制的地址。Git开始将仓库下载到本地。

3.4 在VS Code中添加.gitignore插件

因为Unity项目的文件较多，不是所有文件都有版本管理的必要。因此我们需要为Github指定一个忽略的列表。在插件面板输入git，安装相应插件。在该插件页面，输入，可以将unity项目不必要的文件屏蔽掉。

3.5 提交和push

VS Code自带了git管理工具。在git页面点击可以实现代码版本的本地提交。在按钮的弹出菜单中选择push可以将本地仓库代码推送到远程仓库。

4 后记

4.1 结论

殊途同归是软件开发中的常见现象。使用Github客户端和VS Code进行版本管理各有优势。前者简单直观，通过图形化界面可以方便地进行分支管理，但需要在开发过程中不同软件之间切换，略显麻烦。后者与VS Code紧密结合，可以无缝地实现代码的提交和更新，但是需要用户对命令行操作比较熟悉，难度稍高。
