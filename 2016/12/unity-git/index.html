Win系统下用Github或VS Code实现Unity项目版本管理

发表于 2016-12-18 | 分类于 Unity开发

Unity项目进行版本管理时，需要借助一些外部工具。目前比较流行的版本管理平台是Github。本文探讨Win系统下用Github和VS Code对Unity项目进行版本管理的方法。

1 前言

1.1 需求

VS Code与Unity的集成可见上文，本文不再赘述。通过Github进行版本管理已经非常流行，本文将就Win系统下用Github和VS Code开发Unity项目进行探讨。依托Github进行版本管理主要有两个途径，一个是使用Github的win客户端对代码直接管理，一个是通过VS Code进行集成管理。以下分述。本文默认已注册Github账户。

1.2 术语解释

  • Git：是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。

  • Github：是一个面向开源及私有软件项目的托管平台，因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub。

1.3 版本说明

Win 10，Unity 5.4，VS Code1.8.0，Git 2.11.0 windows.1，Github Desktop 3.3.3.0

2 使用Github客户端进行版本管理

2.1 下载安装Github win客户端

从这里下载最新的客户端并安装。需要注意的是，这里获取的是下载器，需要二次下载安装。如果二次下载遇到错误，可以择时再试。

Ashampoo_Snap_2016.12.18_22h26m55s_001_

2.2 登陆Github账户

打开Github客户端，在右上角点击齿轮打开菜单选择Option，在Accounts中点击+ Add account按钮，输入已注册好的Github账户名和密码。

Ashampoo_Snap_2016.12.19_08h14m54s_001_

2.3 生成并发布Github代码仓库

点击左上角的+号按钮，弹出添加代码仓库的面板，选择Create，输入仓库名称（Name），选择Unity项目的文件夹（Browse），点击Git ignore的下拉框，输入Unity，筛选出适合Unity的Git忽略文件，最后点击Create repository，即可在本地生成仓库，并将项目文件添加到本地仓库。点击右上角的publish按钮，将本地仓库发布到远程账户。

Ashampoo_Snap_2016.12.19_08h31m30s_005_

2.4 更新仓库

当对仓库中的文件进行增删改操作后，可以在客户端页面先admit本地提交，再通过Sync同步push到远程端。其他的操作与Github的常规操作无异，可参考相关资料。

3 使用VS Code进行版本管理

3.1 下载并安装Git

假设Unity项目已经设置好了与VS Code的连接。

在这里下载Git for windows文件并安装，可以根据具体需求设置相关的选项，也可所有选项默认。

Ashampoo_Snap_2016.12.19_08h41m11s_007_

3.2 生成仓库

在Github网站点击Create设置新仓库。

Ashampoo_Snap_2016.12.19_08h43m20s_009_

进入代码仓库，复制当前仓库的地址。

Ashampoo_Snap_2016.12.19_08h44m53s_010_

3.3 将仓库下载到本地

在Unity项目文件夹，点击右键选择Git Bash Here。

Ashampoo_Snap_2016.12.19_08h47m09s_011_

在弹出的Git Bash命令行窗口中，输入git clone以及刚刚复制的git repository地址。Git将仓库下载到本地。

Ashampoo_Snap_2016.12.19_08h54m05s_013_MINGW64--c-work-New Unity Project 1test

3.4 在VS Code中添加.gitignore插件

打开VS Code。因为Unity项目的文件较多，不是所有文件都有版本管理的必要。因此我们需要为Github指定一个忽略的列表。在扩展面板输入gitignore，选择CodeZombie版本的插件并安装。

Ashampoo_Snap_2016.12.19_09h31m12s_014_

按F1键，VS Code顶部出现命令行，输入add gitignore后回车，再输入unity，可以将unity项目中不必要的文件屏蔽掉。

Ashampoo_Snap_2016.12.19_10h13m02s_016_

Ashampoo_Snap_2016.12.19_10h13m59s_017_

VS Code中还有许多Git的扩展工具，不妨一试。

3.5 提交和push

VS Code自带了git管理工具。在Git面板点击对勾按钮可以实现代码版本的本地提交。在…按钮的弹出菜单中选择push或Sync可以将本地仓库代码推送到远程仓库。

Ashampoo_Snap_2016.12.19_10h16m10s_019_

4 结论

条条大路通罗马，不论是用Github的客户端还是用VS Code的Git集成工具，都能够实现对Unity项目的版本管理。这其中一个非常重要的步骤是要确定.gitignore文件中对Unity项目文件的筛选。

使用Github客户端和VS Code进行版本管理各有优势。前者简单直观，通过图形化界面可以方便地进行分支管理，但需要在开发过程中不同软件之间切换，略显麻烦。后者与VS Code紧密结合，可以无缝地实现代码的提交和更新，但是需要用户对命令行操作比较熟悉，难度稍高。
