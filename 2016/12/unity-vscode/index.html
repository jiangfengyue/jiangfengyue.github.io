Win系统下VS Code与Unity联合开发（2016版）

发表于 2016-12-18 | 分类于 Unity开发 | 阅读次数

在Window系统中，习惯了VS的编程环境，在开发Unity项目时，自然而然想利用最新的VS Code来实现集成开发。本文将详细阐述VS Code和Unity的相关设置，手把手教大家建立VS Code中的Unity调试环境。

1 前言

1.1 需求

因Unity自带的MonoDevelop版本低，调试不方便，而我又习惯win系统下的VS的开发模式，所以还是选择最新的VS Code与Unity的集成开发。

1.2 相关术语

VS Code：是微软出品的跨平台语言编辑软件，只提供轻量级的代码编辑和调试功能，可以通过众多的扩展包实现对多种语言的支持和更丰富的语言编辑功能。与VS Studio的区别在于，后者是IDE，能够具备完整的语言编辑、编译、调试和生成的功能；前者只能提供基本的语言编辑和调试功能。

MonoDevelop ：是个适用于Linux、Mac OS X和Microsoft Windows的开放源代码集成开发环境，主要用来开发Mono与.NET Framework软件。MonoDevelop 集成了很多Eclipse与Microsoft Visual Studio的特性，像是 Intellisense、版本控制还有 GUI 与 Web 设计工具。另外还集成了GTK#GUI设计工具（叫做Stetic）。目前支持的语言有Python、Vala、C#、Java、BOO、Nemerle、Visual Basic .NET、CIL、C与C++。

1.3 版本说明

本文用的软件环境是win10，Unity 5.4，VS Code 1.8，VS code assest，Unity debugger，c# extention

2 安装VS Code

2.1 下载VS Code

在https://code.visualstudio.com/Download可以下载最新的VS Code软件。

2.2 安装VS Code

打开安装程序，一路默认安装即可。VS Code Win版只有x86平台的，没有专门64位系统的安装包。

3 安装Unity插件

3.1 下载VS Code插件

打开需要VS Code调试的Unity项目，从Unity Assest Store里查找VSCode（注意这里搜索的是没有空格的VSCode），该插件免费，直接导入默认安装即可。此时你的Unity项目文件夹Assest中就多了一个VS Code文件夹，里面是用于与VS Code成成集成的插件。

3.2 更改设置

在Unity菜单Edit中选择Preferrence，弹出的对话框中选择最下方的VS Code Tab，后上所有选项，点击生成目录。

3.3 链接打开VS Code

在Unity的G菜单栏中选择Open with VS Code，将开启VS Code界面。

4 安装VS Code插件

4.1 安装Unity debugger

选择VS Code左侧面板最下面的extention按钮，会出现相应的extention面板。在上方的输入框输入unity能够筛选出Unity相关的插件，都点击安装。

4.2 打开debug模式

点击extention上方的debug按钮，出现debug面板，点击上方的齿轮按钮，在弹出的选项中选择Unity debugger。此时若出现launch.jason文件如图所示，即表示设置成功。

4.3 安装c#等插件

同样在extention面板中搜索c#、，安装相应的插件。

5 开发调试

5.1 在Unity中点击任意代码，都能打开VS Code

5.2 在编辑器中对代码进行编辑。开发中能够实现比mono更丰富的提示功能

5.3 在Unity开启的前提下，点击debug面板的绿色三角按钮，或者直接F5键，即可开启VS Code的调试模式，会出现以下提示。

5.4 切换回Unity，进入游戏模式

5.5 在VS Code中设置断点，能够对代码进行调试。相应的变量、线程都在debug面板中显示。

6 结论

感谢等大神的维护，让我们能够在VS Code中方便自如地开发Unity项目。通过与其他插件的融合，能够显著提高开发效率。在此过程中如果遇到相关问题，可以尝试以下做法：

  • 1.若出现.Net CLI无法定位的问题时，可以到官网下载.net core sdk并安装
  • 2.若能下断点但无法单步调试时，可以尝试下载安装最新版的monosdk，从github上获取最新的debug后解压到目录
  • 3.若不能正常进入debug模式，可以删除Unity项目目录中的.vscode文件夹，再重复4.2步骤即可
